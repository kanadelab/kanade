########################################################################
#
#
#　奏でる日常の音色　サーフェイス アサイン　サンプル ファイル
#
#
########################################################################

#
#　「奏でる日常の音色」には特定のサーフェイスを別のサーフェイスに置き換えることで、
#　マスターシェルと異なる番号のサーフェイスを持つシェルを使用する機能を搭載しています。
#　この機能を使うことで、サーフェイスの枚数が違うシェルでも使用できるようになります。
#

#
#　・使い方
#
#　　サーフェイス アサイン ファイルを以下の書式でシェルのフォルダに「assign.txt」という名前で配置します。
#　　「assign.txt」が配置されている場合にアサイン機能が自動的に動作します。
#

#
#　・書式
#
#　　・コメント
#
#　　　'#'（半角シャープ）ではじまる行は、コメントとしてみなされ、使用されません。
#
#
#　　・宣言をしない場合
#
#　　　以下の「サーフェイスの置換宣言」「サーフェイス実装宣言」をしない場合には、
#　　　未実装サーフェイスとみなされ、自動的にデフォルトサーフェイス番号に置換されます。
#　　　サーフェイス番号が１０を除く０〜９９９のメインキャラクターのサーフェイスは０番に、
#　　　サーフェイス番号が１０００以降のサブキャラクターのサーフェイスは１０番に置き換わります。
#　　　このアサイン ファイルを有効にした場合は\s[0]が\s[5]になり、\s[6]はそのままですが、
#　　　他のサーフェイスはすべて\s[0]で表示されます。
#　
#　　・サーフェイスの置換宣言
#
#　　　「その番号のサーフェイスを別の番号のサーフェイスで置き換えたい」ときに使います。
#　　　書式は「（置き換え前のサーフェイス番号）,（置き換え後のサーフェイス番号）」です。
#　　　↓の行の#を外すと実際に動作して、\s[0]を\s[9]に置き換えます。
#0,9
#
#
#　　・サーフェイス実装宣言
#　　　
#　　　「その番号のサーフェイスは用意してあるのでそのままで良い」ときに使います。
#　　　書式は「（実装したサーフェイス番号）」です。
#　　　↓の行の#を外すと実際に動作して、\s[5]は\s[5]のまま表示します。
#5
#
#
#　　・特殊設定宣言
#
#　　　シェル側から設定するイベントなどの設定です。
#　　　例えばスカートがめくられたのサーフェスがないのにスカートめくりイベントが発生するのを抑制するなど。
#　　　書式は「（設定名）,（1:有効、0:無効）」です。
#　　　宣言をしない場合のデフォルトはすべて「1」です。
#　　　↓の行の#を外すとスカートめくりイベントが発生しなくなります。
#flip_up_skirt,0
#
#
#
#　　・使用可能な特殊設定宣言
#　　　rool_up_skirt		スカートたくし上げイベントの発生の有無を設定します。
#　　　flip_up_skirt		スカートめくりイベントの発生の有無を設定します。
#　　　kiss			キスイベントの発生の有無を設定します。
#
#

#
# ・注意
#
#　少しでも書式に外れた部分があると、正常に動作しません。（空白の挿入もNG）
#　ただし、何も入力していない行は認識されないので、行間をあけることはできます。
#
#

#
#　・よくわからないときは
#
#　　よくわからない場合は直接質問してください。
#	 ブログやメールなどでお気軽にどうぞ。
#　　アサイン ファイルの作成を依頼して頂いても構いません。
#　　
#　　ただし、初回だけはシェルをリロードしないと動かないことがあります。
#　　（シェル変更時にシェルのフォルダを取得しているため）
#

#
#　・ファイルの配布について
#
#　　サーフェイス アサイン ファイルを同梱したシェルを配布して構いません。
#　　一次配布元の了承があれば、サーフェイス アサイン ファイルを同梱したフリーシェルなどを
#　　「奏でる日常の音色」用追加シェルとして二次配布して構いません。
#　　その他特に制限はありません。サーフェイス アサイン ファイルはファイルを作った人のものです。
#
